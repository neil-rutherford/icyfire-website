IcyFire's Vulnerability Disclosure Program (VDP)

Disclosure

Please note that this program does not allow disclosure. You may not release information about vulnerabilities found in this program to the public.

In scope

www.icy-fire.com/*

IcyFire is starting this program with our primary public-facing web application on icy-fire.com. As the program progresses, more targets will be added to the program.

Testing is only authorized on targets listed as in-scope. Any domain/property of IcyFire not listed above is out-of-scope and is not included in this program.

Scanning activity

Please do not use automated vulnerability scanners on this program. Custom scripts and fuzzing tools are permitted, but if using them, please keep your traffic to 6 requests per second or less. Please avoid using automated tools unless for targeted, specific testing, and then only at less than 6 requests per second.

Non-qualifying vulnerabilities

When reporting vulnerabilities, please consider (1) attack scenario/exploitability, and (2) security impact of the bug. The following issues are considered out-of-scope:

  • Clickjacking on pages with no sensitive actions
  • Cross-Site Request Forgery (CSRF) on unauthenticated forms or forms with no sensitive actions
  • Attacks requiring MITM or physical access to a user's device
  • Previously known vulnerable libraries without a working Proof of Concept
  • Comma Separated Values (CSV) injection without demonstrating a vulnerability
  • Any activity that could lead to the disruption of our service (DoS)
  • Content spoofing and text injection issues without showing an attack vector / without being able to modify HTML/CSS
  • Rate limiting or brute force issues on non-authentication endpoints
  • Missing best practices in Content Security Policy
  • Missing HttpOnly or Secure flags on cookies
  • Missing email best practices (invalid, incomplete or missing SPF/DKIM/DMARC records, etc.)
  • Vulnerabilities only affecting users of outdated or unpatched browsers
  • Software version disclosure / Banner identification issues / Descriptive error messages or headers (e.g. stack traces, application or server errors)
  • Tab nabbing
  • Open redirect - unless an additional security implication can be demonstrated
  • Self XSS
  • Promo code abuse (e.g. ordering multiple times using the same promo code)
  • Abuse of promotional offers and referral codes
  • Promo code enumeration, abuse of our promotional offers and referral codes
  • Able to retrieve user's public information
  • Username / email enumeration
  • Social engineering of users, IcyFire staff, or contractors
  • Bugs in third-party authentications (attacks specifically against our implementation are fine)
  • Results from automated tools without any manual confirmation
  • Any action that interferes with a user's privacy, security, or experience
  • Clear Text / HTTP Basic Authentication
  • Internal Path disclosure
  • Blind XSS, Blind SSRF with no actual exploitation
  • Subdomain takeover

Safe harbor

When conducting vulnerability research according to this policy, we consider this research to be:

  • Authorized in accordance with the Computer Fraud and Abuse Act (CFAA) (and/or similar state laws), and we will not initiate or support legal action against you for accidental, good faith violations of this policy;
  • Exempt from the Digital Millennium Copyright Act (DMCA), and we will not bring a claim against you for circumvention of technology controls;
  • Exempt from restrictions in our Terms of Service that would interfere with conducting security research, and we wait those restrictions on a limited basis for work done under this policy; and
  • Lawful, helpful to the overall security of the Internet, and conducted in good faith.
  • You are expected, as always, to comply with all applicable laws.
